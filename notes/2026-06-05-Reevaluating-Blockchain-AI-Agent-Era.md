### 性能 / 费用

## 三、成本 vs 收益：移动中的临界点

### 过去：成本压过收益

```
收益       成本
  ▲         ▲
  │    ┌────┤
  │    │    │
  │    │    │  ← 链上金融这类强依赖去中心化的应用值得
  │    └────┤  大量中间地带应用停留在临界点之前
  └─────────┘
```

### 现在：AI Agent 移动临界点

```
收益       成本
  ▲         ▲
  │  ┌──────┤
  │  │      │  ← 收益增加（Agentic 场景更多跨系统需求）
  │  │      │  成本降低（工具化、流程化）
  │  └──────┘
       ↓
    更轻的选型决策
```

**推论：AI Agent 正在移动 Web3 选型的临界点**，过去不值得上链的应用，现在有了新的成本收益比。

## 四、用不用区块链，是系统能力选择

> 重点不是谁更先进，而是场景是否需要**共享、可验证的状态和规则**。

| 维度 | 中心化方案 | 区块链方案 |
|------|-----------|-----------|
| **成本** | 成本和复杂度更低 | 规则公开，结果可验证 |
| **交付** | 单一主体内交付直接 | 多方协作，信任成本摊薄 |
| **适用场景** | 单方控制、不需外部验证 | 多方共享状态、需可审计 |

### AI Agent 时代的新权衡

Agentic 场景下会出现更多**跨系统、跨服务、跨主体**的任务：

- **状态怎么共享？** — 多个 Agent 协作时需要共同的状态视图
- **权限怎么约束？** — Agent 的操作权限需要可编程、可审计

区块链正是为这两个问题而生的基础设施。

## 五、复杂性分析：本质 vs 附属

### 用复杂性分层定位 Agent 的作用

| 复杂性类型 | 来源 | 内容（投票 Demo 对照） |
|-----------|------|----------------------|
| **本质复杂性** | 来自要解决的问题 | 信任边界（合约/用户/前端各自承担什么）、投票规则设计、状态归属 |
| **附属复杂性** | 来自解决方式 | CI/CD 配置、测试框架、部署脚本、工具链编排 |

### Agent 的作用

- **处理本质复杂性** — 与 AI Agent 交互式澄清需求，展开路径、比较优势，帮助定位真正需要决策的问题
- **大幅减少附属复杂性** — Demo 阶段把开发环境、工具权限和验收标准放到位，让 Agent 接管实现、调试与验证

> 高度依赖资深经验的部分，正在被工具化、流程化。开发者保留判断，Agent 帮助展开判断，把工程工作推进到可验证结果。

## 六、链上投票 Demo

### 开发流程全景（5 个典型步骤）

```
确认需求 → 实现 → 调试 → 部署 → 合约升级
                                     ↑
                           安全 Review（生产环境必选，本次未覆盖）
```

### 1. 确认需求（处理本质复杂性）

让 Agent 和开发者成为设计伙伴，围绕以下问题持续追问：

- 投票规则是什么？（一人一票 / 权重投票 / 委托投票？）
- 信任边界在哪？（合约负责什么、前端负责什么？）
- 状态归属？（投票结果存链上还是链下？）
- 异常路径？（重复投票 / 截止时间 / 投票撤回？）

输出：设计文件 + Issues + Plan

### 2. 实现（让 Agent 根据计划执行）

使用 **Ralph loop / Codex 目标模式**：

- 开发者定义目标
- Agent 自主拆解、实现、调试
- 交付前自我 review（implementation notes）
- 覆盖边界情况、错误处理、Gas 优化

### 3. 调试 & 4. 部署 & 5. 合约升级

- Agent 辅助调试（Error → 分析 → Fix → Retest 循环）
- 自动化部署脚本生成
- 合约升级方案（代理模式 / 数据迁移）

## 七、Conflux 区块链

本次 Demo 使用的底层链。

- **名称：** Conflux（树图）
- **类型：** 高性能 Layer 1
- **上线时间：** 2020 年 10 月 29 日主网上线
- **核心创新：** 基于树图（Tree-Graph）共识机制——在保持去中心化与安全性的前提下实现高吞吐与低延迟
- **定位：** 可信、安全、无边界的 Layer 1

---

## 学习收获

1. **AI Agent 移动了上链决策的临界点** — 过去算不过账的中间地带应用，现在有了新的 ROI
2. **区块链的本质不是"更先进"而是"能力选择"** — 需要共享、可验证的状态和规则时再上链
3. **Agent 做附属复杂性，人做本质复杂性** — 开发者保留判断权，Agent 辅助展开和落地
4. **Conflux 的树图共识**是 Layer 1 层面的创新，适合对吞吐量有要求的 Web3 × AI 应用
