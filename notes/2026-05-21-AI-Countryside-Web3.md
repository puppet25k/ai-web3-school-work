## 核心框架

**AI + Web3 的核心不是发币，而是经济基础设施。**

```
AI 负责：理解与决策
Web3 负责：身份、支付、结算和审计
```

### 五大方向

1. **去中心化 AI 基础设施** — 算力、推理、模型服务
2. **AI 链上分析工具** — 资金流、地址画像、风险识别
3. **智能钱包和安全助手** — Agent 钱包、权限管理
4. **交易所和 DeFi 智能风控** — 交易模拟、风险预警
5. **机构风控层** — 合规、审计

---

## AI Agent + Web3 实现原理

```
用户给出目标 → LLM 拆解任务 → AgentKit 选择链上 action
→ Provider 签名和广播 → 链上结果返回给 agent
```

**关键问题：如何把 LLM 的工具调用连接到真实链上动作。**

### 代表性项目

| 项目 | 核心看点 |
|------|----------|
| **Coinbase AgentKit** | 真正的关键不是"AI 会操作钱包"，而是"用户怎么安全授权" |
| **MoonPay Agents** | Agentic commerce 不是能对话，而是能付款和完成购买 |
| **Privy Agentic Wallets** | 智能钱包 + 权限管理 |

## 去中心化 AI 基础设施

### Bittensor (TAO)

```
Subnet 定义任务和评分规则
       ↕
Miner 提供 AI 服务 ↔ Validator 评估输出质量
       ↕
链上共识汇总评分
```

- **核心看点：** AI 服务如何用 token 发现价格和组织生产
- **TAO 解释（evo swift）：** 用自己的电脑提供算力，节点老师打分，电脑按分数收取代币 → 算力被收集给 Agent 使用

### Render Network

- 真实资源供需网络，而不是"AI 套壳"
- [官网](https://rendertoken.com)

### Akash Network

- 开发者能否用更开放的方式购买计算资源
- [官网](https://akash.network)

## AI 链上分析

### 实现原理

```
采集多链交易数据 → 地址聚类和标签 → 归因到实体和钱包集群
→ 计算资金流和行为画像 → 生成搜索、告警和分析视图
```

**核心看点：**
- 链上透明性和隐私之间的张力
- AI 把链上分析从"写查询"变成"提问题"
- 链上研究从"看价格"走向"看资金和地址行为"
- 代表项目：Nansen

---

## 风险边界（Risk Boundary）

1. **AI Agent 一旦开始做事，就会引出账户、支付、授权和追责问题**
2. **项目对应具体环节** — 每个项目都不是独立的，它在回答 Agent 经济栈里的一个具体问题
3. **Token 不是价值本身**

> **AI + Web3 的价值不是给 AI 套 token，而是让软件主体进入可授权、可结算、可审计的经济系统。**
